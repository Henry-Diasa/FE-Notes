#### 项目角色

- PM产品经理
- UE视觉设计师
- FE前端开发
- RD后端开发
- CRD移动端开发
- QA测试人员

#### 完整项目流程

需求分析(各个角色)  => 技术方案设计(FE RD CRD) => 开发(FE) =>联调(FE RD CRD)=>测试(FE QA)=>上线(FE) 

**需求分析**

- 了解背景
- 质疑需求是否合理
- 需求是否闭环
- 开发难度如何
- 是否需要其他支持
- 不要急于给排期

**技术方案设计**

- 求简，不过度设计
- 产出文档
- 找准设计重点
- 组内评审
- 和RD CRD沟通
- 发出会议结论

**开发**

- 如何反馈排期（buffer）
- 符合开发规范
- 写出开发文档
- 及时单元测试
- Mock API
- Code Review

**联调**

- 和RD CRD技术联调
- 让UE确定视觉效果
- 让PM确定产品功能

**PM加需求怎么办**

- 走需求变更流程即可
- 如果公司有规定，则按规定走
- 否则，发起项目组和leader的评审，重新评估排期

 **测试**

- 提测发邮件，抄送项目组
- 测试问题要详细记录
- 有问题及时沟通，QA和FE天生信息不对称

**上线**

- 上线之后及时通知QA回归测试
- 及时同步给PM和项目组
- 如有问题，及时回滚。先止损，在排查问题
